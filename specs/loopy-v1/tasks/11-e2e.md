## scripts/live-eval.ts  (run manually: `npx tsx scripts/live-eval.ts`)

Purpose: measure REAL proposal quality with the actual `claude -p` runner.
- Build 3 synthetic digest sets in-memory (write nothing to LOOPY_HOME; use a
  temp dir for any state the engine insists on — set LOOPY_HOME to a mkdtemp):
  - PLANT A (babysitting): 5 sessions, each with user msgs "check ci" /
    "is the deploy green?" + `gh run watch` tool calls.
  - PLANT B (recurring_task): 4 sessions across 4 Mondays, "update deps and
    run audit" pattern.
  - DECOY: 1 session with a one-off task ("rename this file") that must NOT
    become a candidate.
- Run the engine once over the combined digests with `defaultRunner()` from
  engine (real claude CLI). Print a report:
  `PLANT A: FOUND/MISSED (type, confidence)`, same for B,
  `DECOY: CORRECTLY IGNORED / FALSELY PROPOSED`,
  every candidate's summary + impactEstimate, every warning.
- Exit code: 0 only if both plants found AND decoy not proposed. Exit 1 otherwise.
- Guard: if `claude` CLI missing (spawn ENOENT) → print "claude CLI not found,
  skipping live eval" and exit 0 (CI safety).
